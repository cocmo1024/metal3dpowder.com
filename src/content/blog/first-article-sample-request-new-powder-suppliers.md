The fastest way to create supplier noise is to treat a new powder source like an approved recurring supplier on day one. The better route is usually a controlled first article sample request: one lot, one defined scope, one clear evaluation path.

That approach helps the buyer learn whether the supplier can support the real requirement before the program depends on them commercially.

## What a first article sample request should do

A first article sample request is not a casual trial ask. It should define:

- exact alloy designation
- intended AM process
- target PSD or feedstock window
- sample quantity
- required certificate pack
- package format
- destination and timing

This is still a buying action, not a lab-only exercise. The supplier needs enough clarity to quote and ship a representative lot rather than a generic sample.

If that scope is not yet defined, start with the main [metal powder RFQ checklist](/posts/Alloys/metal-powder-rfq-checklist/) before asking a new supplier for anything.

## Why a first lot should be narrow and explicit

The purpose of a sample lot is to test the supplier fit, not to force the project into premature production behavior. That is why first article requests are usually better when they are:

- narrow in quantity
- explicit in documentation
- honest about project stage
- tied to a defined evaluation plan

Trying to combine supplier onboarding, qualification, and replenishment into one poorly defined order usually creates confusion rather than speed.

## What the buyer should evaluate on the first lot

The first lot should help answer three questions:

1. can the supplier ship the requested alloy and PSD scope cleanly
2. is the certificate and traceability package strong enough
3. does the lot behave acceptably in the intended workflow

That means the review should cover both the paperwork and the receiving event. The related [incoming inspection checklist](/posts/Alloys/incoming-inspection-checklist-am-powder-lots/) is the right companion once the first shipment arrives.

## Sample requests should not pretend to be production orders

Many buyers create avoidable friction by asking a new supplier for production-grade certainty while still operating with prototype-grade ambiguity. If the project is still in a learning phase, say so. If it is crossing toward controlled supply, say that too.

The point is not to make the sample request "look bigger." The point is to let the supplier quote and prepare the right first lot for the real stage of the program. The [prototype vs production specification guide](/posts/Alloys/prototype-vs-production-powder-specification/) explains that transition in more detail.

## What to request beyond the powder itself

For a first article lot, it is usually reasonable to request:

- lot identity and batch reference
- certificate pack matching the sample scope
- package identity that can be checked on receipt
- any specific receiving or release notes relevant to the program

If the long-term workflow will require a more formal document handoff, define that early rather than waiting until the supplier has already become operationally important. The dedicated [prototype-to-qualification handoff guide](/posts/Alloys/prototype-to-qualification-powder-document-handoff/) is built for that step.

## The practical takeaway

A first article sample request is the cleanest way to evaluate a new powder supplier without creating false production expectations. Keep the request controlled, align it to the real program stage, and make sure the first lot is judged on documentation quality and release discipline as well as on material identity.
