A powder quote is only part of the buying decision. The certificate package behind that quote is what tells the buyer whether the lot can actually enter the intended workflow.

For additive manufacturing powders, the document review should not stop at a chemistry table. Buyers need enough information to connect the lot back to the project specification, the batch history, and the incoming inspection plan.

## What a buyer usually needs to see

The exact package depends on the project, but most serious RFQs eventually need some combination of:

- chemistry certificate
- oxygen or interstitial data where relevant
- PSD report
- batch or lot identification
- packaging and traceability information
- shipment or export documents

That package may be light for development work and much heavier for regulated, aerospace, or production-oriented programs.

If the order is international, the technical certificate pack should also line up with the broader [export document checklist](/posts/Alloys/export-document-checklist-metal-powder/) so the shipment file stays coherent.

## Chemistry and interstitial control

The chemistry certificate is the first line of review. For some alloys, that means confirming the expected major elements. For others, especially titanium and some nickel routes, interstitial control is just as important as the headline alloy identity.

That is why the buyer should not approve a lot simply because the alloy name on the label looks correct. The certificate has to match the actual chemistry window the project expects.

## PSD and test clarity

A PSD number on its own is not enough. Buyers should know the reported range and, ideally, how the PSD is defined in the release process. If the RFQ is tied to a narrow LPBF window, the PSD certificate becomes a central commercial document rather than a supporting extra.

This matters because the same nominal alloy can still behave differently if the size distribution drifts. For many AM programs, PSD is one of the first checks that separates a usable lot from a risky one.

## Morphology, flow, and density

Not every supplier package will include the same secondary data, but morphology, flowability, and apparent density should not be treated as optional if the project is serious. These factors influence recoating behavior, feed stability, and consistency across builds.

For a prototype lot, buyers may accept a lighter package. For qualification or repeat production, the supporting quality data usually needs to be stronger.

## Traceability is a commercial issue, not just a quality issue

Traceability is what keeps a powder lot usable after it leaves the supplier. If the lot cannot be clearly tied to the certificate, the package, and the purchase record, the buyer may have difficulty releasing it internally.

That is why lot number, packaging identity, and shipment consistency should be reviewed as part of procurement rather than only as a receiving task.

## Red flags during certificate review

The most common red flags are:

- vague or incomplete batch identification
- a PSD report that does not match the RFQ
- chemistry data that is technically valid but not aligned with the internal requirement
- missing interstitial data where the project clearly needs it
- unclear packaging traceability

These are not always signs of a bad supplier. Sometimes they simply show that the RFQ was incomplete. But they are still reasons to pause before approval.

Once the shipment arrives, the certificate review should be tied directly into a formal [incoming inspection checklist](/posts/Alloys/incoming-inspection-checklist-am-powder-lots/) so the lot is judged against the actual receiving and release workflow rather than against the paperwork alone.

## The practical takeaway

A certificate package should help the buyer decide whether the lot is commercially and technically releasable. If the document set cannot answer that question, the safest move is to clarify it before shipment rather than after the powder reaches the machine room.

For controlled end uses, pair certificate review with the right downstream standard of proof. Medical and dental programs should reference the [medical and dental documentation guide](/posts/Alloys/medical-dental-powder-documentation-guide/), while flight-related programs should start from the [aerospace qualification checklist](/posts/Alloys/aerospace-powder-qualification-checklist/).
